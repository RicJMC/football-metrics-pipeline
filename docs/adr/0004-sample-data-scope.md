# ADR-0004: Synthetic sample data only, no scraped data in Git

- **Status:** Accepted
- **Date:** 2026-05-30
- **Deciders:** Repository owner

## Context

The repository needs *some* in-tree data so that:

- Characterization tests are reproducible without a network round-trip.
- A fresh clone can run `npm run pipeline:sample` and see a real CSV.
- Recruiters and reviewers can evaluate the pipeline without scraping
  FBref themselves.

But the repository must not redistribute scraped FBref tables (see
[ADR-0002](0002-quarantine-generated-data.md)).

## Decision

The only data committed to the repository is **synthetic**, lives under
`sample-data/`, and is small enough to read at a glance. Values are chosen
to exercise the schema and the pipeline branches, not to look like realistic
football statistics.

Current scope:

- `sample-data/etl-phase2/` — happy path. Three synthetic players,
  one season, one team each. Drives the stage 2–5 characterization test.
- `sample-data/edge-cases/` — three small variants:
  - `missing-fields` — players with `null` or absent metric fields.
  - `multi-team` — one player split across two teams in the same season.
  - `filtered-out` — a player below the `minutes_90s < 3` stage-2 threshold.
- *(Planned)* `sample-data/etl-stage1/data/` — mocked `data/` tree for
  exercising `playerStats01_unicos.js`.

All fixtures are generated by deterministic scripts under `scripts/internal/`
so they can be regenerated rather than hand-edited.

## Alternatives considered

- **Subset of real scraped data with player names redacted.** Still legally
  ambiguous (FBref ToU), and harder to keep deterministic across regenerations.
- **External `*-sample-data` repository.** Adds friction to `git clone`
  without solving any real problem for a project of this size.
- **Generate fixtures at test time from a seed.** Loses snapshot diff
  visibility in PRs; regressions become harder to inspect.

## Consequences

- Positive: Zero exposure to FBref's terms-of-use boundary.
- Positive: Fixtures are version-controlled, diffable, and regenerable.
- Positive: Total tracked sample-data footprint is under 100 KB.
- Negative: Sample values are not realistic; they cannot be used for
  analytical demonstrations, only for pipeline correctness.
- Follow-up: When stage 1 fixture lands, document the schema of the mocked
  `data/<id>-<league>/<season>/<category>/*.json` tree.
